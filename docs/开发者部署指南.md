## 3. 部署流程（网页端）

网页端更新后，所有浏览器用户**刷新页面**即可获得最新版本，无需任何操作。这是最常用的部署方式。

### 3.1 完整流程（5 分钟）

```bash
# 步骤 1：进入项目目录
cd werewolf-online

# 步骤 2：确认代码无误（本地可先测试）
npm run dev          # 本地启动，浏览器打开 http://localhost:5173 验证

# 步骤 3：构建前端
npm run build:client
# 产物在 client/dist/，确认构建成功

# 步骤 4：一键部署到云服务器
node deploy-to-server.js <服务器密码>
# 例如: node deploy-to-server.js MyPassword123
```

> 这一步会自动完成：连接SSH → 上传34+个文件 → npm install → 重启pm2服务

### 3.2 验证部署

```bash
# 浏览器打开
http://210.16.170.144:4000

# 或命令行验证
curl http://210.16.170.144:4000/api/status
# 返回: {"success":true,"version":"2.11.0",...}
```

## 4. 一键部署命令

### 4.1 最常用命令

```bash
# 修改了前端代码 → 构建 + 部署
npm run build:client && node deploy-to-server.js <密码>

# 只改了服务端代码 → 直接部署（无需重新构建前端）
node deploy-to-server.js <密码>

# 改了前端但没有改服务端 → 构建 + 部署
npm run build:client && node deploy-to-server.js <密码>
```

### 4.2 部署脚本做了什么

| 步骤 | 操作 | 详情 |
|------|------|------|
| `[1/4]` | 收集文件 | 扫描 `server/src/`（21 个）、`client/dist/`（12 个）、`download/`（4 个） |
| `[2/4]` | 连接服务器 | SSH 到 `root@210.16.170.144:22` |
| `[3/4]` | 上传文件 | SFTP 逐个上传到 `/opt/veilland-server/` |
| `[4/4]` | 重启服务 | `npm install --production` + `pm2 restart veilland-server` |

## 5. 手动部署（备用方案）

如果自动脚本出问题，可以手动操作：

### 5.1 打包

```bash
cd werewolf-online

# 构建前端
npm run build:client

# 打包所有需要上传的文件
tar -czf veilland-deploy.tar.gz \
  server/src/ \
  server/package.json \
  client/dist/ \
  download/
```

### 5.2 上传

```bash
# 用 SCP 上传到服务器
scp veilland-deploy.tar.gz root@210.16.170.144:/root/
```

### 5.3 服务器端操作

```bash
# SSH 登录
ssh root@210.16.170.144

# 解压覆盖
cd /opt/veilland-server
tar -xzf /root/veilland-deploy.tar.gz --strip-components=0

# 复制前端文件
cp -r client/dist/* dist/

# 安装依赖
npm install --production

# 重启服务
pm2 restart veilland-server

# 查看状态
pm2 status
pm2 logs werewolf-server
```

## 6. 客户端更新指南

你的项目有 **3 种客户端**，各自的更新机制不同：

### 🌐 网页端（浏览器）

> **零操作！** 用户刷新浏览器自动获得最新版本。

网页端代码打包在 `client/dist/`，通过服务器静态托管。每次部署后，用户刷新页面时浏览器会重新请求 `index.html` 和新的 JS/CSS 资源（文件名含 hash，天然解决缓存问题）。

### 💻 PC 桌面端（Electron / Windows EXE）

**适用场景：** 改动了 `electron/`、`package.json` 版本号、或想让用户离线也能玩

```bash
# 步骤 1：打包新版 EXE
npm run dist:win
# 产物: release/Werewolf_Setup_2.11.0.exe

# 步骤 2：复制到下载目录
cp release/Werewolf_Setup_2.11.0.exe download/

# 步骤 3：更新版本号（package.json 中的 version 字段）
# 改为新版本号，如 "2.12.0"

# 步骤 4：部署到服务器
npm run build:client && node deploy-to-server.js <密码>
```

**用户如何更新：**
- 打开游戏 → 自动检测版本 → 弹出更新提示
- 或访问 `http://210.16.170.144:4000` → 下载PC端 → 下载最新 EXE
- 下载后重新安装即可（便携版直接替换）

### 📱 Android 客户端

```bash
# 步骤 1：构建前端
cd client
npm run build

# 步骤 2：同步到 Android 项目
npx cap sync

# 步骤 3：用 Android Studio 打开 client/android，构建 APK

# 步骤 4：复制 APK 到 download 目录
cp client/android/app/build/outputs/apk/debug/app-debug.apk download/werewolf.apk

# 步骤 5：部署
cd ..
node deploy-to-server.js <密码>
```

## 7. 常见问题排查

### 7.1 服务器无法访问

```bash
# 检查服务器状态
ssh root@210.16.170.144 "pm2 status"

# 查看日志
ssh root@210.16.170.144 "pm2 logs werewolf-server --lines 30"

# 检查端口
ssh root@210.16.170.144 "ss -tlnp | grep 4000"
```

### 7.2 部署后页面没变化

1. **浏览器缓存** → 按 `Ctrl + Shift + R` 强制刷新
2. **构建失败** → 检查 `npm run build:client` 是否有报错
3. **文件未上传** → 检查部署脚本输出，确认 38 个文件全部上传

### 7.3 服务器报 SyntaxError

服务器 Node.js 版本是 **v20.20.2**，本地是 v24。某些语法可能不兼容：

| 语法 | Node 20 | Node 24 |
|------|---------|---------|
| `import ... with { type: 'json' }` | ❌ 需用 `assert` | ✅ |
| 类字段声明 | ✅ | ✅ |
| 顶层 await | ✅ | ✅ |

**建议：** 部署前在本地用 Node 20 测试，或使用 `nvm use 20` 切换版本。

### 7.4 部署脚本连接超时

```bash
# 先手动测试 SSH 连接
ssh root@210.16.170.144

# 如果连不上，检查：
# 1. 网络是否可达（ping 210.16.170.144）
# 2. 服务器是否在线
# 3. 密码是否正确
```

### 7.5 PM2 进程反复重启

```bash
# 查看错误日志
ssh root@210.16.170.144 "pm2 logs werewolf-server --err --lines 20"

# 手动启动查看报错
ssh root@210.16.170.144
cd /opt/veilland-server
node index.js    # 直接运行看报错信息
```

## 8. 部署脚本说明

### `deploy-to-server.js`

这是核心部署脚本，位于项目根目录。

```javascript
// 配置区（如需更换服务器，改这里）
const HOST = '210.16.170.144';    // 服务器IP
const PORT = 22;                   // SSH端口
const USER = 'root';               // 用户名
const REMOTE_BASE = '/opt/veilland-server';  // 服务器部署路径
```

### 上传的文件范围

| 来源 | 远程路径 | 说明 |
|------|----------|------|
| `server/src/*.js` | `/opt/veilland-server/*.js` | 服务端源码（扁平化，无src目录） |
| `server/src/*/` | `/opt/veilland-server/*/` | 子目录（game/auth/socket等） |
| `server/package.json` | `/opt/veilland-server/package.json` | 依赖清单 |
| `client/dist/*` | `/opt/veilland-server/dist/*` | 前端静态文件 |
| `download/*` | `/opt/veilland-server/download/*` | 客户端安装包 |

### PM2 管理命令

```bash
# 在服务器上
pm2 status                  # 查看所有进程
pm2 logs werewolf-server    # 实时日志
pm2 restart veilland-server # 重启
pm2 stop werewolf-server    # 停止
pm2 start index.js --name veilland-server  # 首次启动
pm2 save                    # 保存进程列表（重启后自动恢复）
pm2 startup                 # 设置开机自启
```
